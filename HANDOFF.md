# CampusNotesAI Handoff

## 프로젝트 개요
- 프로젝트명: `CampusNotesAI`
- 목적: 대학생 대상 AI 기반 노트 앱 캡스톤 MVP
- 현재 스택: `Vite + React + pdfjs-dist`
- 백엔드: 없음
- AI 연동: 없음
- 현재는 프론트엔드 데모/프로토타입 중심 상태

## 현재 구현 상태 요약
### 1. 메인 홈 화면
- 메인 화면과 노트 상세 화면이 분리되어 있음
- 홈 화면은 삼성 노트 느낌의 구조로 정리된 상태
- 좌측 사이드바 메뉴
  - `모든 노트`
  - `즐겨찾기`
  - `휴지통`
  - `폴더`
- `폴더`가 기본 메인 화면
- 폴더 화면에서는 상단에 폴더 카드, 아래에 노트 카드가 보임
- 폴더에 속하지 않은 노트는 별도로 표시됨
- 폴더/노트 데이터는 현재 mock 데이터 기반

### 2. 메인 화면 기능
- `노트 작성` 기능 있음
- `폴더 생성` 기능 있음
- 노트 우클릭 메뉴
  - `삭제`
  - `즐겨찾기 추가/해제`
  - `공유`
- 폴더 우클릭 메뉴
  - `이름변경`
  - `삭제`
- 노트 삭제 시 휴지통으로 이동
- 폴더 삭제 시 폴더와 폴더 내부 노트가 함께 휴지통으로 이동
- 삭제 전 확인 다이얼로그 표시됨

### 3. 노트 상세 화면
- 상세 화면은 더 이상 바깥에 `AI 패널 / 학습자료 패널`이 따로 있는 구조가 아님
- 현재는 하나의 큰 학습자료 섹션(`note-study-shell`) 안에 아래 요소가 모두 들어감
  - 상단 가로 툴바
  - 좌측 AI Assistant
  - 우측 PDF/필기 영역
- 상단 툴바
  - `메인으로`
  - 노트명
  - 필기/타이핑/지우개/캡처/PDF 업로드 등 버튼
- 좌측 AI Assistant
  - 학습자료 섹션 안쪽에 포함됨
  - 접기/펼치기 가능
  - 드래그로 너비 조절 가능
  - 최대 폭은 화면 절반 정도
  - 툴바와 경계 없이 최대한 붙도록 레이아웃 조정됨
- AI 리사이즈 핸들
  - 시각적으로는 거의 보이지 않음
  - 실제 드래그 가능한 hit area는 유지함

### 4. PDF 뷰어 / 필기 영역
- `pdfjs-dist`로 실제 PDF 페이지를 렌더링함
- 페이지가 세로로 쌓여 있고 스크롤로 페이지 이동함
- `Ctrl + 스크롤` 시 PDF 영역만 확대/축소됨
- 확대/축소는 CSS scale 기반
- 기본 데모 PDF는 `public/4_Maximum likelihood learning.pdf`

### 5. 필기 / 메모 기능
- 손글씨 모드 지원
- 타이핑 모드 지원
- 지우개 모드 지원
- 영역 캡처 모드 지원
- 손글씨는 PDF 위에 오버레이 방식으로 구현됨
- 타이핑은 텍스트 박스 방식이 아님
- 각 페이지는 큰 `textarea` 하나를 사용하지만, 클릭 규칙은 줄 단위로 커서 이동하게 커스텀 처리됨

### 6. 현재 타이핑 규칙
- 클릭한 `x` 좌표는 무시됨
- 클릭한 `y` 좌표만 줄 높이 기준으로 계산됨
- 클릭 위치가 `5.5번째 줄` 같은 상태면 `5번째 줄`로 스냅됨
- 클릭 후 커서는 항상 해당 줄의 시작점(0열)으로 이동함
- 즉 사용자가 중간을 클릭해도 그 줄 맨 앞에서 타이핑 시작

### 7. Enter / Backspace 페이지 이동 규칙
- `Enter`
  - 현재 페이지 마지막 줄이 아니면 같은 페이지에서 줄바꿈
  - 현재 페이지 마지막 줄에서만 다음 페이지 첫 줄로 이동
- `Backspace`
  - 같은 페이지에서 첫 줄이 아니면 한 줄 위로 이동
  - 현재 페이지 첫 줄에서만 이전 페이지로 이동
  - 이전 페이지로 이동할 때는 이전 페이지의 기존 입력 상태와 관계없이 항상 마지막 줄 시작점으로 이동
- 이 동작은 사용자가 원하는 노트 필기 흐름 기준으로 여러 번 조정해서 현재 맞춘 상태

### 8. Undo / Redo
- 손글씨, 타이핑 모두 `undo/redo` 대상
- 현재 보고 있는 페이지만이 아니라 노트 전체 편집 기록 기준으로 동작
- 지원 단축키
  - `Ctrl+Z`: undo
  - `Ctrl+Shift+Z`: redo
  - `Ctrl+Y`: redo
- 타이핑 중 `Tab`은 다음 요소 이동이 아니라 들여쓰기로 동작

### 9. 영역 캡처 / 이미지
- `영역 캡처` 모드에서 자유선택 경로를 그릴 수 있음
- 선택한 영역을 이미지로 생성함
- 생성된 캡처는 현재 페이지 아래 미리보기 카드로 쌓임
- 이미지 업로드 기능 있음
- 카메라 입력 기능 있음
- 아직 구현 안 된 것
  - 캡처 결과를 AI Assistant에 바로 전달
  - 캡처 결과 복사/삭제 버튼
  - 캡처 결과를 AI 문맥에 직접 연결

### 10. 저장 / 복원
- 일부 편집 상태는 `localStorage`에 저장됨
- 현재 저장/복원 대상
  - 타이핑 메모
  - 손글씨 stroke
  - redo stroke 상태
  - 노트별 현재 보고 있던 페이지 인덱스

- 노트 상세 화면 레이아웃은 최근에 크게 바뀌었음
  - 예전의 바깥 AI 패널 구조가 아님
  - 지금은 내부 학습자료 섹션 구조임
- AI 리사이즈 핸들은 실제보다 훨씬 얇아 보이게 처리해 둔 상태
