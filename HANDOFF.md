# CampusNotesAI Handoff

## 프로젝트 개요
- 프로젝트명: `CampusNotesAI`
- 목적: 대학생 대상 AI 기반 노트 앱 캡스톤 MVP
- 현재 스택: `Vite + React + pdfjs-dist`
- 백엔드: 없음
- AI 연동: 없음
- 현재는 프론트엔드 데모/프로토타입 중심 상태

## 현재 구현 상태 요약
### 1. 메인 홈 화면
- 메인 화면과 노트 상세 화면이 분리되어 있음
- 홈 화면은 삼성 노트 느낌으로 정리된 상태
- 좌측 사이드바 메뉴
  - `모든 노트`
  - `즐겨찾기`
  - `휴지통`
  - `폴더`
- `폴더`가 기본 메인 화면
- 폴더 화면에서는 상단에 폴더 카드, 아래에 노트 카드가 보임
- 폴더에 속하지 않은 노트는 별도로 표시됨
- 폴더/노트 데이터는 현재 mock 데이터 기반

### 2. 노트 상세 화면
- 노트에 들어가면 홈 화면의 좌측 메뉴 사이드바는 사라짐
- 대신 상세 화면은 아래 구조
  - 좌측: `AI Assistant` 사이드바
  - 우측: 학습자료(PDF) + 필기 영역
- 좌측 AI Assistant 사이드바
  - 접기/펼치기 가능
  - 드래그로 너비 조절 가능
  - 화면의 절반 정도까지 확장 가능
  - 너무 작게 줄이면 접힘 상태로 전환
- 우측 학습자료 영역
  - `pdfjs-dist`로 실제 PDF 페이지를 렌더링함
  - 페이지가 세로로 쌓여 있고 스크롤로 페이지 이동함
  - 상단 이전/다음 버튼 없음
  - 현재는 스크롤로 페이지 전환하는 구조

### 3. 필기/메모 기능
- 손글씨 모드 지원
- 타이핑 모드 지원
- 지우개 모드 지원
- 영역 캡처 모드 지원
- 손글씨는 PDF 페이지 위에 직접 필기하는 느낌으로 구현됨
- 타이핑은 텍스트 박스 방식이 아니라 페이지 위에 줄글처럼 바로 쓰는 방식
- 페이지 바깥 빈 공간에는 필기 불가
  - 이건 현재 구조상 이미 해결된 상태

### 4. 확대/축소
- 버튼식 확대/축소는 제거됨
- `Ctrl + 스크롤` 시 브라우저 전체가 아니라 학습자료 영역만 확대/축소되도록 구현됨
- 확대/축소는 PDF 재렌더링이 아니라 CSS 스케일 기반이라 이전보다 훨씬 부드러움

### 5. Undo / Redo
- 손글씨, 타이핑 모두 `undo/redo` 대상
- 현재 보고 있는 페이지만이 아니라 노트 전체 편집 기록 기준으로 동작
- 예를 들어
  - 1페이지 손글씨 작성
  - 2페이지 손글씨 작성
  - 2페이지에서 `Ctrl+Z`
  - 가장 최근 작업부터 전체 노트 기준으로 되돌아감
- 지원 단축키
  - `Ctrl+Z`: undo
  - `Ctrl+Shift+Z`: redo
  - `Ctrl+Y`: redo
- 타이핑 중 `Tab`은 다음 요소 이동이 아니라 들여쓰기로 동작하도록 수정됨

### 6. 영역 캡처
- `영역 캡처` 모드에서 마우스로 자유선택 경로를 그릴 수 있음
- 선택한 영역을 잘라서 이미지로 생성함
- 생성된 캡처는 현재 페이지 아래 미리보기 카드로 쌓임
- 아직 구현 안 된 것
  - 캡처 결과를 AI Assistant에 바로 전달
  - 캡처 결과 복사/붙여넣기
  - 캡처 결과 다운로드/삭제 버튼

### 7. 저장/복원
- 일부 편집 상태는 `localStorage`에 저장됨
- 현재 저장/복원 대상
  - 타이핑 메모
  - 손글씨 stroke
  - redo stroke 상태
  - 노트별 현재 보고 있던 페이지 인덱스

## 지금 상태에서 특히 중요한 UX 메모
- 손글씨 튕김 문제는 여러 번 수정해서 현재는 많이 안정화된 상태
- 스크롤 전환도 많이 개선된 상태
- AI 사이드바 드래그 동작은 사용자 기준으로 "완벽하게 마음에 들지는 않지만 일단 보류" 상태
  - 나중에 다시 다듬을 가능성 있음
